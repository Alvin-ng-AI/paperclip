# Brainstorm Skill

Facilitate structured, high-quality brainstorming sessions that produce genuinely useful ideas — not generic filler.

## Core Principles

- **Diverge first, converge later** — generate quantity before filtering for quality
- **No self-censoring** — include bold, unconventional ideas alongside safe ones
- **Always cluster** — group ideas into categories so the user can navigate them
- **Always recommend** — after generating, pick your top 3 and say why
- **Match energy** — quick 5-min ideation vs. deep strategic session are different modes

## Workflow

### 1. Clarify (if needed)
Before generating, check if you have enough context:
- **Topic**: What are we brainstorming about?
- **Goal**: What outcome does the user want from these ideas?
- **Constraints**: Budget, time, audience, platform, tone?
- **Volume**: Quick burst (5–10 ideas) or exhaustive exploration (20–50+)?

If context is clear from the message, skip straight to generating — don't ask unnecessary questions.

### 2. Choose a Mode

| Mode | When to use | Output |
|---|---|---|
| **Quick Burst** | Casual ask, simple topic | 8–12 ideas, 1–2 sentences each |
| **Deep Dive** | Strategic decision, complex topic | 20–30 ideas, clustered by theme |
| **Forced Connections** | Stuck or need breakthrough ideas | Combine unexpected domains |
| **SCAMPER** | Improving something existing | Substitute / Combine / Adapt / Modify / Put to other use / Eliminate / Reverse |
| **How Might We** | Problem-solving | Reframe problem as HMW questions first |

Default to **Quick Burst** unless the request signals depth is needed.

### 3. Generate Ideas

Structure output as:

```
## 💡 [Topic] — Brainstorm

### [Category A]
- **Idea name** — one-line description
- ...

### [Category B]
- ...

---
### 🏆 Top 3 Picks
1. **[Idea]** — why this one
2. **[Idea]** — why this one
3. **[Idea]** — why this one
```

### 4. Offer Next Steps

After delivering ideas, offer to:
- Develop any idea further
- Score ideas against constraints
- Explore a completely different angle

## Quality Checks

Before delivering:
- [ ] Ideas span full spectrum (safe → bold → wild)?
- [ ] Every idea is distinct — no near-duplicates?
- [ ] Ideas are specific enough to act on?
- [ ] Top 3 recommendation included?
- [ ] Categories aid navigation?
