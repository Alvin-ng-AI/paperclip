# Ideation Frameworks Reference

## SCAMPER (for improving existing things)
- **S**ubstitute — what can be replaced?
- **C**ombine — what can be merged?
- **A**dapt — what can be borrowed from elsewhere?
- **M**odify/Magnify — what can be changed, enlarged, or emphasized?
- **P**ut to other use — how else could this be used?
- **E**liminate — what can be removed?
- **R**everse/Rearrange — what if it worked backwards or differently?

## Six Thinking Hats (for group or structured thinking)
- 🟡 **Yellow** — optimistic, benefits, value
- ⚫ **Black** — caution, risks, problems
- 🟢 **Green** — creativity, alternatives, new ideas
- 🔵 **Blue** — process, control, next steps
- ⚪ **White** — data, facts, information
- 🔴 **Red** — emotions, gut feeling, intuition

## Jobs To Be Done (for product/service ideas)
Frame the problem as: "When [situation], I want to [motivation], so I can [outcome]."
Then generate ideas that solve the job better, cheaper, or differently.

## Blue Ocean vs Red Ocean
- **Red Ocean** — compete in existing market, fight for share
- **Blue Ocean** — create uncontested new market space
Ask: what can we eliminate, reduce, raise, or create vs. industry standard?

## Hook Model (for content/product engagement ideas)
- **Trigger** — what prompts the behaviour?
- **Action** — simplest behaviour in anticipation of reward
- **Variable Reward** — what's the unpredictable payoff?
- **Investment** — what makes the user put something in?

## How Might We (HMW)
Reframe problems as opportunities:
"How might we [achieve goal] for [audience] despite [constraint]?"
Generate 5–10 HMW questions before generating solutions.

## Random Stimulus
Pick an unrelated object/industry and force connections:
"How would [Tesla / a sushi restaurant / a kindergarten teacher] solve this problem?"
